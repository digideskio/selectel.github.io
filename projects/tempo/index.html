Tempo

Библиотека Tempo предназначена для парсинга и форматирования времени на Erlang. Она реализует интерфейс к функциям strptime и strftime из стандартной библиотеки libc.

Описание

Tempo обеспечивает интерфейс для библиотек strptime/strftime, написанным на C, обеспечивающий корректную обработку форматов времени на Erlang.

Все экспортированные функции могут выдать ответ badarg в случае, если данные на входе заданы некорректно.

Аргумент type может принимать следующие значения:

  • unix - метка времени UNIX, положительное целое число, означающее число секунд, прошедших с 1 января 1970 года;
  • now - erlang:now/0:
  • datetime - calendar:datetime

Аргументом Format для любой из экспортированных функций может быть либо binary(), состоящая из описателей поля и текстовых символов, совместимых с strptime/strftime, либо один из следующих атомов: iso8601, rfc1123, rfc2822. В последнем случае будет использован предопределенный формат.

В 32-битных системах функции семейства format могут возвращать ошибку {error, time_overflow} в случае, если 32-битное значение превышает допустимый предел. Минимальное значение для таких систем составляет {{1901,12,13},{20,45,52}} и максимальное - {{2038,1,19},{3,14,7}}.

Типы данных

datetime_type()

datetime_type() = unix | now | datetime

datetime_value()

datetime_value() = unix_timestamp() | erlang:timestamp() | calendar:datetime()

format()

format() = binary() | iso8601 | rfc1123 | rfc2822

unix_timestamp()

unix_timestamp() = float()

Функции

format/2 форматирует кортеж {Type, Datatime} в соответствии с установленным форматом
format/3 форматирует дату и время в соответствии с установленным форматом
format_datatime2 вспомогательная функция, аналогичная format/3
format_now/2 вспомогательная функция, аналогичная format/3
format_unix/2 вспомогательная функция, аналогичная format/3
parse/2 парсит кортеж {Type, Binary} в соответствии с указанным форматом, возвращает кортеж ok/error вместе с датой и td временем в формате, определяемом атомом Type
parse/3 парсит Binary в соответствии с указанным форматом, возвращает кортеж ok/error вместе с датой и временем в td формате, определяемом атомом Type
parse_datatime2 вспомогательная функция, аналогичная parse/3
parse_now/2 вспомогательная функция, аналогичная parse/3
parse_unix/2 вспомогательная функция, аналогичная parse/3

Подробное описание функций

format/2

format(Format::format(), X2::{datetime_type(), datetime_value()}) -> {ok, binary()} | {error, invalid_time} | {error, time_overflow}

Эквивалентна format(Format, Datetime, Type).

format/3

format(Format::format(),Datetime::datetime_value(), Type::datetime_type()) -> {ok, binary()} | {error, invalid_time} | {error, time_overflow}

format_datetime/2

format_datetime(Format::format(), Datetime::calendar:datetime()) -> {ok, binary()} | {error, invalid_time} | {error, time_overflow}

Эквивалента format(Format, Datetime, datetime).

format_now/2

format_now(Format::format(), X2::erlang:timestamp()) -> {ok, binary()} | {error, invalid_time} | {error, time_overflow}

Эквивалентна format(Format, Datetime, now).

format_unix/2

format_unix(Format::format(), Timestamp::unix_timestamp()) -> {ok, binary()} | {error, invalid_time} | {error, time_overflow}

Equivalent to format(Format, Datetime, timestamp).

parse/2

parse(Format::format(), X2::{datetime_type(), binary()}) -> {ok, datetime_value()} | {error, format_mismatch}

Эквивалентна parse(Format, DatetimeType, Binary).

parse/3

parse(Format::format(), Bin::binary(), Type::datetime_type()) -> {ok, datetime_value()} | {error, format_mismatch}

parse_datetime/2

parse_datetime(Format::format(), Bin::binary()) -> {ok, calendar:datetime()} | {error, format_mismatch}

Эквивалентна parse(Format, Binary, datetime).

parse_now(Format::format(), Bin::binary()) -> {ok, erlang:timestamp()} | {error, format_mismatch}

Эквивалентна parse(Format, Binary, now).

parse_unix(Format::format(), Bin::binary()) -> {ok, unix_timestamp()} | {error, format_mismatch}

Эквивалентна parse(Format, Binary, timestamp).

Пример использования 

1> {ok, Bin} = tempo:format(iso8601, {now, now()}).
{ok,<<"2013-01-05T13:29:18+0400">>}
2> tempo:parse(iso8601, {datetime, Bin}).
{ok,{{2013,1,5},{13,29,18}}}

По умолчанию используется три формата времени: iso8601, rfc1123, и rfc2822.

Можно использовать любой формат, соответствующий требованиям libc: 

(tempo_dev@localhost)1> {ok, Bin} = tempo:format(<<"%A, %Y-%d-%m">>, {now, now()}).
{ok,<<"Thursday, 2012-07-06">>}

Известные ограничения и проблемы

При парсинге дат до 1900 года под OS X возникает ошибка {error, format_mismatch}.
